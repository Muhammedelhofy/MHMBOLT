Seeing a 401/403 sync error? Tap Fix Sync — it re-links to a fresh cloud bin, re-uploads everything (history + captain profiles), and copies the new share link for you to send to the other devices.
🔑 Private Cloud Key (fixes recurring 403)
The built-in key is shared by everyone using this dashboard, so its free monthly quota runs out and throws 403 for all. Use your own free key for a private quota: 1) make a free account at jsonbin.io → 2) API Keys → copy your Master Key → 3) paste it below & Save → 4) tap Fix Sync to move your data onto it, then share the new link (it carries the key, so other devices auto-configure).
🟢 Supabase (recommended — no 100KB limit, scales with your fleet)
JSONBin's free plan caps each upload at 100KB (~2–3 weeks of data). Supabase free stores years of data and full history for the whole team. One-time setup: 1) sign up free at supabase.com → New project → 2) open SQL Editor and run the snippet below (creates the table + opens team access) → 3) Project Settings → API → copy the Project URL and the anon public key → 4) paste both below → Save → Test → then Fix Sync. The share link carries both, so other devices auto-configure.

create table if not exists fleet_data (
  id text primary key,
  data jsonb,
  updated_at timestamptz default now()
);
alter table fleet_data enable row level security;
create policy "team access" on fleet_data
  for all using (true) with check (true);
